# 협업 규칙 (한국어)

- 사용자와의 모든 대화(설명, 질문, 작업 계획, 오류 설명, 완료 보고)는 한국어로 작성한다. 단, 터미널 명령어와 코드 자체는 영어로 작성해도 된다.
- 초보자도 이해할 수 있도록 쉽게 설명한다. 전문 용어를 쓸 때는 간단히 풀어서 설명한다.
- 작업을 시작하기 전에 변경할 파일과 작업 계획을 먼저 알려주고 진행한다.
- 파일 삭제, 데이터 삭제, 배포(deploy), git push 작업은 실행 전에 반드시 사용자 승인을 받는다.
- 이 프로젝트는 TypeScript를 사용한다.
- Next.js App Router 구조(`src/app`)를 유지한다.
- Firebase 설정값과 API 키는 항상 환경변수로 관리하고, 코드에 하드코딩하지 않는다.
- 비밀키(secret key)와 토큰은 `NEXT_PUBLIC_` 접두사가 붙은 환경변수에 넣지 않는다. `NEXT_PUBLIC_`은 브라우저에 노출되므로 서버 전용 비밀값(예: `ANTHROPIC_API_KEY`)에는 사용하지 않는다.
- 코드를 변경한 뒤에는 `npm run build`로 오류가 없는지 확인한다.
- 기존 코드를 무조건 지우지 않고, 먼저 분석해서 재사용 가능한 부분을 최대한 활용한다.
- 답변 마지막에는 변경된 파일 목록과 다음 할 일을 한국어로 간단히 요약한다.
